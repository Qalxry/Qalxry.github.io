问题一：'_POSIX_SEM_VALUE_MAX' was not declared in this scope

CMake输出：

这好像是由于这个GCC的C++20实现有点BUG，将cxx_std_20 改为cxx_std_17 C++17及以下即可。 询问Claude得到回复如下：（虽然不确定是否正确）

这个错误提示在Windows环境下编译时,不能找到_POSIX_SEM_VALUE_MAX的定义。

_POSIX_SEM_VALUE_MAX是POSIX系统中的一个宏,用于定义semaphore的最大计数值。但是在Windows环境中,并没有这个定义。

很可能是C++标准库的代码原本基于POSIX平台编写,移植到Windows的时候没有适配导致的错误。

要解决这个问题,有以下几种方法:

  1. 在Windows环境下,不要直接使用_POSIX_SEM_VALUE_MAX,可以定义一个兼容值。修改C++标准库实现,根据平台宏对_POSIX_SEM_VALUE_MAX进行隔离。

  2. 在编译命令中定义_POSIX_SEM_VALUE_MAX,使其成为一个空宏。

  3. 尝试更新编译器或标准库版本,使用更兼容的实现。

  4. 根据编译器文档报告此兼容性问题。

  5. 联系原库维护者进行跨平台适配。

  6. 隔离使用非兼容接口的代码,避免污染Windows环境。

  7. 考虑用更通用的线程同步原语替换semaphore。

总之,这主要是一个跨平台兼容性问题。需要区分平台适配相关接口,或者更新编译器和库的版本来获得修复。

问题三：Asio缺少Windows相关系统库文件链接

CMake输出信息：

网上（Google、Bing、Github、StackOverFlow）难以找到答案，因为报错信息有点太了，询问Claude得到答案：

这个链接错误的原因是目标可执行文件在链接过程中无法解析 Windows sockets API 所需要的符号,包括:

  • WSAStartup, WSACleanup - Winsock 初始化和清理函数

  • WSAGetLastError, WSASetLastError - 获取和设置错误码的函数

  • AcceptEx, GetAcceptExSockaddrs - 接受连接的函数

  • closesocket, ioctlsocket 等 - 套接字操作的函数

  • ntohl, htonl, ntohs, htons - 网络字节序转换函数

  • select, WSASend, WSARecv 等 - 套接字I/O操作函数

导致这个错误的常见原因:

  1. 没有链接 Windows socket 库 ws2_32.lib,它包含上述 API 的实现。

  2. 依赖的网络库没有自动链接所需的系统库。

  3. 编译设置没有指定使用 Winsock。

  4. vcpkg动态链接设置不正确,应使用动态CRT链接。

  5. 没有根据平台预定义正确隔离连接代码。

解决方法是:

  1. 显式链接 ws2_32.lib。

  2. 确认依赖库已自动链接必要的系统库。

  3. 定义相关宏以开启Winsock支持。

  4. 使用动态链接选项并检查编译器设置。

  5. 根据Windows平台预定义隔离特定实现。

明确添加对 ws2_32.lib 的链接可以解决大部分此类问题。
